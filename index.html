Mac上Intellij IDEA使用Maven创建完整Java Web项目

发表于 2016-09-05   |   分类于 Java   |     |   阅读次数

1 基础环境配置

1.1 JDK安装

版本选择

对于MacOS来说，目前有两种JDK：

  • 苹果原生JDK：最高版本1.6
  • Oracle的最新JDK：目前最高1.8

我选择的是Oracle最新的1.8，安装包名字为’jdk-8u102-macosx-x64.dmg’，链接如下

http://www.oracle.com/technetwork/java/javase/downloads/jdk8-downloads-2133151.html

如果需要苹果原生JDK，请访问：

https://support.apple.com/kb/dl1572?locale=zh_CN

动态获取JDK路径

在Mac下，可以使用下面的方法，动态获取这个环境变量，对于安装了JDK1.8的电脑：
(注意各位机子上的JDK版本，修改对应的1.8为1.7、1.6等)

/usr/libexec/java_home -v 1.8

此时在我的终端下会输出如下路径：

/Library/Java/JavaVirtualMachines/jdk1.8.0_102.jdk/Contents/Home

环境变量配置

编辑(vi)文件’~/.zshrc’(zsh用户)或’~/.bash_profile’(bash用户)，加入下面两行

export JAVA_HOME=/usr/libexec/java_home -v 1.8

export PATH=${JAVA_HOME}/bin:$PATH

1.2 maven安装

安装Maven

Maven有两种安装方式：

  • 直接去官网下载Maven安装包，然后解压，配置环境变量
  • 利用brew工具安装

注意Maven基于Java，对JDk的版本有要求，比如JDK1.6就不支持Maven3.3.1及以上版本了。具体见如下官网：

https://maven.apache.org/docs/history.html

我直接使用方便快捷的brew安装，命令如下：

brew install maven

等待安装完成，输入：

mvn -v

如有版本信息出现安装maven成功。

配置文件路径

  • 全局配置文件：/usr/local/Cellar/maven/3.3.9/libexec/conf/settings.xml
  • 用户配置文件： ~/.m2/settings.xml

一般编辑全局配置文件即可。

配置需要的jar包()

不使用Maven的时候，我们需要去官网下载相关的jar包，然后导入到项目中。现在使用maven的话，就不需要上网找jar包了。Maven所做的工作就是自动把你需要的jar包下载到本地，然后关联到项目中来。maven的所有jar包都是保存在几个中央仓库里面的，其中一个最常用的是Maven Repository，即，你需要什么jar包，它就会从仓库中拿给你。那么如何告诉maven需要什么jar包呢？我们看看工程目录，能找到一个pom.xml文件（这个文件在刚创建好项目时就已经自动打开了），maven就是靠它来定义需求的

我们可以看到这个文件包含了我们之前定义的本项目的gropId等信息，这些信息是该项目的标识，我们不要去改动它们。重点看标签，翻译过来是’依赖’的意思，也就是说把对每个包的需求都称为一个依赖，定义在中。在每个中，你需要提供的是所需jar包的groupId、artifactId、version这三个必要信息。比如上面我们看到Maven自动引入了一个junit包，格式如下：
